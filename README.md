### ПОДХОД
Гипотеза: Самый эффективный источник — отраслевые мероприятия и списки.
Компании, которые участвуют/выступают в качестве спикеров в форумах по контакт-центрам или рейтингах сервиса,
почти гарантированно имеют большую команду поддержки и открыты к публичности.

### Источник данных
Форум контакт центр 2025 и ccwf

### Валидация доменов и ИНН - DaData

### Определение признаков поддержки

**Каналы связи:**
* Email поддержки: паттерны support@, help@, info@
* Онлайн-чат: наличие виджетов (jivo, tawk, chatra, crisp)
* Мессенджеры: Telegram, WhatsApp, Viber
* Формы обратной связи: поиск <form> с атрибутами contact/feedback

**Структурные признаки:**
* Раздел поддержки: ссылки с текстом "поддержка", "support", "помощь", "контакты"
* FAQ/база знаний: поиск разделов FAQ, "вопросы-ответы", knowledge base
* 24/7: ключевые слова "24/7", "круглосуточно", "24 часа", "без выходных"

### Формула оценки минимального размера команды:
+20 за 3+ вакансии поддержки

+10 за 2 вакансии

+5  за 1 вакансию

+15 за разные роли

+20 за круглосуточную работу (24/7)

+15 за сменную работу

+10 за многоуровневую поддержку (L1/L2)

+10 за наличие чата + телефона

+15 за упоминания высокой нагрузки

**Итоговый** score = Σ(бонусы)

score ≥30 → 18+ человек

score ≥20 → 15+ человек

score ≥15 → 12+ человек

score ≥10 → 10+ человек

score <10 → 0 человек

### Объединение данных
По ИНН - первичный ключ дедупликации
По нормализованному названию - удаление ООО/АО, приведение к нижнему регистру

### РЕЗУЛЬТАТЫ
**Что удалось**
* Практически автоматический сбор 50 компаний с поддержкой ≥10 человек
* Двойная валидация через сайты и вакансии HH.ru
* Обработка 86 ИНН с сохранением формата
* Параллельный парсинг источников

**Ограничения**
* Ручной ввод официальных сайтов, автоматический поиск требует дополнительной интеграции
* Отсутствие Уровня A - не найдено прямых упоминаний численного размера команд
* Ограничения HH API - rate limiting (3 сек между запросами)
* Валидация данных - часть email/URL могут быть некорректны

## Запуск
Для запросов к DaData необходим API ключ в .env: DADATA_API_KEY='API-key'
```
git clone git@github.com:elllies/test_task.git
cd test_task/
python -m venv venv
source venv/Scripts/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```
Отдельный запуск получения первичных списков компаний
```
python -m src.collect_seeds
```
Запуск остальных функций
```
python main.py
```
